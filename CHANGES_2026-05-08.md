# Changes — May 8, 2026

## Agent Pipeline & OAuth

### OAuth token scope bug fix
The token endpoint defaulted to `catalog:read` when no `scope` param was sent, silently dropping `orders:create`. Fixed per RFC 6749 §4.4: if no scope is requested, grant all scopes the client is authorized for.

**File:** `api/routes/oauth.py`

### Email verification gate on OAuth client creation
Unverified users could create OAuth clients. Added a 403 guard on `POST /oauth/clients` requiring `is_verified = true`.

**File:** `api/routes/oauth.py`

## Payment Flow Fixes

### Stripe webhook secret not configured
`STRIPE_WEBHOOK_SECRET` was missing from Railway environment variables, causing every `checkout.session.completed` event to be rejected with `{"detail":"Webhook not configured"}`. No transactions were being written and no emails were being sent. Fixed by configuring the webhook endpoint in Stripe Dashboard and adding the signing secret to Railway.

### StripeObject `.get()` crash in session endpoint
`stripe.checkout.Session.retrieve()` returns a `StripeObject`, not a plain dict. Calling `.get()` on it raises `AttributeError`. Fixed by switching to attribute access (`getattr`) throughout `get_checkout_session`.

**File:** `api/routes/payments.py`

### StripeObject `.get()` crash in webhook handler
Same issue in `handle_successful_payment` — `event['data']['object']` is also a `StripeObject`. Fixed with `getattr` throughout metadata extraction.

**File:** `api/routes/payments.py`

### Success page missing Navbar
The `/success` page had no navigation, leaving users stranded after purchase. Added `<Navbar />`.

### Success page: guest CTA improvement
For guest purchases, the primary CTA now says "Create account to view purchases" and links to `/signup?email=<purchase_email>` so the email is pre-filled. Since signup merges the guest user row into the new account, all purchases appear automatically with no extra steps.

### Success page: removed dead error branch
The `error` state was initialized to `''` and `setError()` was never called, making the entire error branch dead code. Removed to clean up the component.

**File:** `web/pages/success.tsx`
