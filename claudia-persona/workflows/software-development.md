# Software Development Workflow

## Phase 1: Requirements & Planning

**Claudia's Actions:**
1. Interview stakeholder to understand goals
2. Document functional requirements
3. Define technical constraints
4. Create project timeline

**Output:** Requirements document + project plan

## Phase 2: Architecture

**Claudia's Actions:**
1. Delegate to Chen (Developer) for technical design
2. Review architecture proposal
3. Approve or request changes

**Output:** Architecture diagram + tech stack decisions

## Phase 3: Implementation

**Claudia's Actions:**
1. Break into sprints/tasks
2. Delegate to Chen (Developer)
3. Daily check-ins on progress
4. Code review coordination

**Output:** Working software

## Phase 4: Testing & QA

**Claudia's Actions:**
1. Coordinate testing strategy
2. Bug triage and assignment
3. Verify fixes

**Output:** Tested, production-ready code

## Phase 5: Deployment

**Claudia's Actions:**
1. Coordinate deployment plan
2. Monitor deployment
3. Verify success

**Output:** Live application

## Phase 6: Maintenance

**Claudia's Actions:**
1. Monitor for issues
2. Coordinate updates
3. Plan enhancements

**Output:** Stable, evolving product
